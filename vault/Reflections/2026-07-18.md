## Session: Cross-device docs across Foundation + kits + website (00:15)

### What worked

- Refused to guess the behavioral steps. The user's redirect was vague
  ("behavioral, not config"); I stated a wrong mental model (mobile =
  capture-only) as a *guess to correct*, and the correction — full sessions
  everywhere via Remote Control + Dispatch — reshaped the entire doc. Guessing
  silently would have shipped wrong instructions.
- Verified feature terminology with the claude-code-guide agent before writing
  setup steps, instead of documenting from memory.
- After the named point-fix, audited the whole doc surface (grep across all 5
  repos) and found the same bug in 8 more files. Fixing what was named would
  have left the stale app name and broken install path everywhere else.

### What didn't

- I committed the first Foundation cross-device section (`dbe3bf6`) before
  auditing the surface — and that commit *introduced* a stale "Claude Code
  desktop app" term in my own new text, which the later audit then had to
  re-fix in the same file. Committing a point fix before checking the whole
  surface (including my own additions against the repo's naming convention)
  meant reopening a file I'd just closed.
- Tried to `Edit` two kit files after only `cat`-ing them via Bash → Edit
  refused ("not read yet"), costing a round-trip. Bash `cat` doesn't satisfy
  the Read-before-Edit requirement.

### Lesson (one sentence, actionable)

Before committing a wording/convention fix, grep the whole surface it belongs
to — sibling files *and* the new text you just wrote — so one commit closes the
issue instead of reopening the same file.

  promote to a MEMORY.md standing lesson only if it repeats.
