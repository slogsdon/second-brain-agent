### What didn't

- I built AND committed the hook before checking whether the platform already
  did the job. The whole value of an `ask`-returning PreToolUse hook depends on
  what Claude Code's *default* permission mode already does — and default mode
  already prompts on Edit/Write with a diff. Section 4 of the handoff even
  listed the permission-mode mechanics; I evaluated the proposed `ask` hook on
  its own internal logic without cross-checking it against the built-in `ask`.
  The user surfaced it in two prods; I should have asked "what does default
  mode already give us here?" during the Section 5 evaluation, before writing
  a line. Result: a build-then-revert round trip and a commit pair that nets to
  zero change.

### Lesson (one sentence, actionable)

Before building a control/mechanism, check whether the tool or platform already
provides it by default (ponytail rung 3–4) — evaluate a proposed feature
against existing built-in behavior, not only against its own internal logic.
