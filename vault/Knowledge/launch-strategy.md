# Loop & Gate — launch strategy

Plan to take the Loop & Gate stack (Foundation + Build/Grow/Accountability
kits + shane.logsdon.io landing page) to a wide audience. Decided model:
**free now, paid later**. Primary goal: **audience → activated adoption**
(reach is a means, not the scoreboard). Structured as a gate: the wide launch
does not fire until a feedback gate passes.

## The wedge (positioning)

Market is flooded with *fully-autonomous* agent frameworks. Differentiator is
**human judgment gates at the decisions that matter** — "autonomy where it's
cheap, judgment where it's expensive." Category: the judgment layer /
human-in-the-loop harness for Claude Code agents. Named enemy: slop shipped by
unchecked loops. One line: "Loop & Gate — your agent does the work, you keep
the judgment." This lane is open — everyone else races toward *more* automatic;
Loop & Gate sells restraint as a feature.

## Phase 0 — feedback gate (must pass before wide launch)

The onboarding cliff kills a plugin faster than weak reach. Phase 0 de-risks
first-run for a cold user on their own machine.

- Recruit 10–20 hands-on testers (warm network + targeted DMs in r/ClaudeCode,
  r/ObsidianMD — not public posts).
- Instrument **activation** = installed Foundation → ran `profile-interview` →
  completed one full loop iteration. Watch where testers stall.
- Cold-install test on Mac AND Windows by someone who isn't the author.

Exit gate (all must hold): ≥60% activate; ≥3 usable testimonials; top-5
friction points fixed; ≥1 tester runs a 2nd session unprompted (retention).

## Where — channels, tiered

1. **Passive discovery (do now, compounds):** submit to
   `anthropics/claude-plugins-community`; aim for `claude-plugins-official`;
   list on claudemarketplaces.com, `ComposioHQ/awesome-claude-plugins`,
   aitmpl.com. Always-on funnels — get listed before launch.
2. **First real users (lead here):** r/ClaudeCode (~4,200 weekly contributors,
   best fit), r/ClaudeAI, r/ObsidianMD (vault-memory + Accountability angle),
   r/SideProject, IndieHackers. Warm communities 2–3 weeks before launch with
   genuine contribution — cold launch posts get flagged.
3. **Launch boards (big swings, AFTER users vouch):** Dev Hunt, Uneed/Fazier,
   then Product Hunt + Show HN (plain text, technical merit, no marketing voice).
4. **Audience (owned, not rented):** build-in-public on X + LinkedIn; the essay
   on blog/dev.to. Add email capture to the landing page NOW — for
   free-now-paid-later, the owned list IS the product; followers don't convert.

## When — sequence

Now→2–4wks: Phase 0 feedback + fix; in parallel submit passive listings, warm
communities, write essay + record demo. Feedback gate passes → publish essay +
demo, tease build-in-public. Launch week (Tue–Thu, US morning): Product Hunt
~9am ET; Show HN ~9h later; **r/ClaudeCode +12h** (best surface, not
r/SideProject); LinkedIn +16h reframed. Wide launch is a moment; adoption is a
grind — passive listings + SEO essay + standing community presence out-earn
launch day over any 90-day window.

## How — highest-leverage assets (ranked)

Existing tease post — ["Building on the Margins"](https://shane.logsdon.io/articles/strategic-insights/building-on-the-margins/)
— is the deliberate top-of-funnel: it already presents the Loop & Gate method
(ten gates, sizing-by-importance, keep/handoff, three kits, honest limits)
bound to LeadSurface as a personal workflow, CTA = follow. Do NOT write a
competing manifesto — it splits SEO and re-tells what's live. Launch assets
convert the warm reader the tease already produced.

1. **60–90s demo video/GIF** of one full loop iteration — the gates stopping
   and asking (the pause is the product). Highest leverage; the tease post has
   no video. Carries PH, Show HN, landing page. Cut a 6s autoplay loop + 75s
   full. See demo shot-list (Build kit, `/loop-and-gate`).
2. **Install tutorial** ("run this in 5 min") — product-first, reader's-agent
   as protagonist, decoupled from LeadSurface, ends in `/plugin install` +
   email capture. The practical companion the story tease deliberately isn't.
   This is the missing written piece — not another essay.
3. **Repurpose the tease post:** add a "want this as a tool? → repos + list"
   block so it feeds installs, and reuse its cold-open as the Show HN text.
4. Optional narrow-angle post (only if a new essay is wanted): one focused
   piece on the single non-obvious idea — the sizing gate / "uniform ceremony
   is theater" — which the tease touches but doesn't own. New angle, not new
   summary.
5. Phase-0 testimonials on the landing page.
6. Honest TAM framing: Pro/Max required — say it up front to filter, not churn.

## Free → paid seam

Name it now, build later. Design principle: core loop + plugins + method stay
free forever (they are the adoption/audience engine — paywalling them kills
it); each paid tier removes a pain the free tier deliberately leaves. The paid
tier = the exact thing the free tier makes users feel the absence of.

Sequencing insight: do NOT start with SaaS. Start with course + done-with-you —
they monetize the audience directly, need no infra, and are the only tiers that
tell you which SaaS to build. Ship sync/team AFTER paying customers show the
pain. SaaS first = guessing.

Options, ranked by fit + willingness-to-pay + build cost:

1. **Paid course / "the method"** (do first) — how to build with gates on AI
   agents; the essay is chapter one. Self-paced ~$150–500 or cohort ~$1–2k.
   Near-100% margin, no infra, best fit with the audience→adoption goal, and it
   reveals what people will pay to have done for them.
2. **Done-with-you setup + custom gates** (do first) — install, wire the vault,
   author domain gates for teams/founders. ~$500–2k/engagement. Fastest cash,
   validates demand, each engagement is product research (sell service before
   SaaS).
3. **Team / multiplayer tier** (build after DWY) — highest ceiling; shared
   vault memory + profiles across a team, gates as approval checkpoints (natural
   fit — a gate already means "a human decides"), aggregated accountability.
   ~$15–30/seat/mo. The real business if there is one.
4. **Managed vault-sync / hosted memory** — sharpest single-pain wedge from the
   cross-device friction in the docs (iCloud gotchas, Android/Windows sync,
   worktree discipline). Free = self-managed; paid = it-just-works across
   devices. Prosumer ~$5–10/mo. Lower ceiling than team; solo on-ramp to it.
5. **Vault analytics / "second brain, quantified"** (later, low build) — reads
   data already produced: gate-pass rates, what you keep deferring
   (Accountability kit), velocity, which gates get rubber-stamped. ~$5–10/mo
   add-on; strong bundled into team/sync, weak alone.
6. **Gate-pack / profile marketplace** (year two) — sell or take a cut of
   community kits, industry gate sets, voice/taste profile packs. Network
   effects, needs install volume first.

Skip: hosted runtime (Anthropic owns that surface — thin resale margin);
crippleware kits (fragments free value, poisons the adoption story);
sponsorship/donations (fine signal, not a business).

Pricing above is speculative — validate in Phase 0 / DWY.

## Metrics

North star: activated installs (ran ≥1 full loop), not stars/upvotes.
Audience: owned-list subscribers, not follower count. Leading:
install→activation rate, D7 retention (ran a 2nd session).

## Biggest risks

Onboarding cliff > reach. Launching before retention exists (leaky bucket).
Treating launch day as the strategy rather than one input to a compounding
system.
